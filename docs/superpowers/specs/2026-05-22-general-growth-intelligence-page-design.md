# 通用增长情报系统页面设计文档

## 1. 产品定位

本产品定位为通用行业的社交媒体增长情报系统，面向内容运营、增长负责人、市场研究、品牌投放和竞品分析团队。用户不需要先理解爬虫参数、数据库表或 AI 配置，而是从业务问题出发：找达人、追内容、盯友商、判断关键词热度。

首版目标不是做炫酷大屏，而是做到业务可用：结果能稳定产出，每个判断都有证据，样本不足时明确提示，用户能基于系统结果执行下一步动作。

核心原则：

- 通用行业，不绑定教育、母婴或任何固定垂类。
- 项目驱动，每个项目承载一个业务研究问题。
- 证据优先，每个达人、关键词、友商异常、内容机会都能回溯到样本。
- AI 辅助配置和解释，用户确认后执行。
- 采集、分析、任务、导出都服务业务工作流，不作为主要体验入口。

## 2. 推荐信息架构

一级导航保留 7 个入口：

1. 今日情报
2. 项目工作台
3. 达人发现
4. 内容追踪
5. 友商监控
6. 关键词热度
7. 设置

下列能力不再作为一级页面，而是合并到业务页面中：

- 增长机会决策：合并到今日情报。
- 数据浏览：合并为项目内的样本与证据，以及各结果的证据抽屉。
- AI 分析：分散到达人、内容、友商、关键词、项目洞察中。
- 后台任务：合并为顶部任务中心和项目历史任务。
- 关键词库：合并为项目研究配置和设置里的模板库。
- 导出中心：合并为各页面的导出动作。

全局固定区域：

- 顶部任务中心：显示运行中任务、失败任务、可重试任务。
- 全局项目选择器：用户可切换当前工作项目。
- 全局新建入口：新建项目、新建采集、新建追踪器、新增友商。
- 证据抽屉：任何结论点击后右侧打开证据详情。

## 3. 页面一：今日情报

### 页面定位

今日情报是用户每天打开系统后的第一个页面，回答三个问题：

- 今天有什么值得做？
- 哪些机会正在变强或变弱？
- 当前数据是否足够可信？

它不应该只是指标看板，而应该是行动面板。

### 目标用户

- 老板或增长负责人：快速看今天有没有机会和风险。
- 运营负责人：决定今天要跟进哪些达人、内容、友商、关键词。
- 分析人员：检查异常是否有证据支撑。

### 首屏布局

首屏分三栏：

- 左侧：今日行动清单。
- 中间：机会队列。
- 右侧：风险与样本质量。

顶部显示：

- 当前项目或全部项目切换。
- 时间范围：今日、近 24 小时、近 7 天。
- 平台筛选：全部、小红书、抖音等。
- 刷新按钮和任务中心入口。

### 核心模块

#### 3.1 今日行动清单

按优先级展示系统建议：

- 立即跟进达人：来自达人发现评分 A 档。
- 立即测试内容方向：来自关键词热度和同类内容增长。
- 查看友商异常：来自友商每日流量突增或内容结构变化。
- 补采关键词：来自样本质量不足。
- 处理任务失败：来自采集诊断。

每条行动卡包含：

- 行动标题。
- 业务原因。
- 影响对象：达人、关键词、友商、内容。
- 优先级：高、中、低。
- 证据数量。
- 建议动作按钮。

按钮示例：

- 查看证据
- 创建追踪器
- 加入候选池
- 补采样本
- 标记已处理

#### 3.2 机会队列

展示来自四个工作台的机会：

- 达人机会
- 内容机会
- 友商变化
- 关键词机会

每张机会卡包含：

- 机会名称。
- 类型标签。
- 平台。
- 机会分。
- 24 小时变化。
- 样本可信度。
- 核心证据摘要。
- 风险标签。

交互：

- 点击卡片打开机会详情抽屉。
- 可以按类型、平台、可信度、状态筛选。
- 可以反馈：有效、观察、误判、已执行。

#### 3.3 风险与样本质量

展示影响判断可信度的问题：

- API key 缺失或失效。
- 平台采集失败。
- 某项目样本不足。
- 某平台近 24 小时无新增数据。
- 友商监控未完成。
- AI Provider 不可用。

每条风险应给出处理动作：

- 去设置 API key。
- 重试任务。
- 调整关键词。
- 降低目标量。
- 暂停该平台判断。

#### 3.4 今日摘要

系统自动生成一句话摘要和分点说明：

- 今天最强关键词是什么。
- 哪个达人最值得跟。
- 哪个友商变化最大。
- 哪个内容模式在冒头。
- 哪些结论样本不足。

摘要必须引用证据，不允许只输出泛泛建议。

### 空状态

无数据时不显示空白图表，而显示三步引导：

1. 创建一个项目。
2. 生成研究配置。
3. 启动首次采集。

### 失败状态

如果数据不足或采集失败，应展示具体原因：

- 缺少 TikHub API key。
- 关键词没有结果。
- 平台暂不可用。
- 任务已失败但可重试。
- 入库成功但后处理失败。

### 需要优化现有功能

现有总览和增长机会决策页应合并。机会分、趋势图、风险提醒保留，但入口从“看图表”改成“执行今日动作”。

### 需要新增功能

- 今日行动清单。
- 统一机会详情抽屉。
- 样本质量总览。
- 人工反馈状态。
- 跨工作台跳转动作。

## 4. 页面二：项目工作台

### 页面定位

项目工作台承载一个业务研究问题。所有关键词、标签、采集计划、样本、AI 洞察和导出都围绕项目组织。

示例项目：

- 宠物食品达人发现
- 新能源车内容机会研究
- 本地生活探店竞品监控
- 美妆新品种草关键词热度

### 页面结构

项目工作台分为左侧项目列表和右侧项目详情。

右侧详情使用 Tab：

1. 项目概览
2. 创建/研究配置
3. 采集计划
4. 样本与证据
5. 洞察与建议
6. 历史任务
7. 导出

### 4.1 项目列表

列表展示：

- 项目名称。
- 项目目标。
- 关注平台。
- 最近采集时间。
- 样本量。
- 健康状态。
- 今日机会数。

筛选：

- 进行中
- 观察中
- 已归档
- 有异常
- 样本不足

操作：

- 新建项目
- 复制项目
- 归档项目
- 进入项目

### 4.2 项目创建向导

通用产品必须弱化表单，使用向导式创建。

步骤一：业务目标

用户选择一个或多个目标：

- 找达人
- 追内容
- 盯友商
- 看关键词热度
- 生成每日情报

步骤二：业务语境

用户输入：

- 行业或产品
- 目标用户
- 典型需求或痛点
- 希望排除的对象
- 重点平台

示例输入：

```text
我想研究宠物主粮在小红书和抖音上的种草达人，排除宠物医院和低价批发账号。
```

步骤三：AI 生成研究配置

系统生成：

- 核心关键词
- 扩展关键词
- 同义词
- 排除词
- 达人画像标签
- 内容模式标签
- 友商识别规则
- 初始采集计划

用户可编辑、删除、补充。

步骤四：采集计划预览

展示：

- 平台
- 采集模式
- 关键词数量
- 预计样本数
- 预计耗时
- 是否采评论
- 是否采达人主页
- 可能失败原因
- API key 和平台能力检查

用户确认后启动。

### 4.3 项目概览

展示项目状态：

- 样本总量
- 达人数量
- 关键词数量
- 友商数量
- 追踪器数量
- 最近采集
- 失败任务
- 样本质量

同时展示四个工作台的项目级入口：

- 查看达人发现结果
- 查看内容追踪
- 查看友商监控
- 查看关键词热度

### 4.4 研究配置

配置分组：

- 业务语境
- 平台范围
- 关键词组
- 排除词
- 标签定义
- 评分权重
- 采集策略

交互：

- AI 补充关键词。
- AI 生成排除词。
- 一键去重。
- 检测过窄或过宽关键词。
- 保存为模板。

关键词类型：

- 核心词
- 扩展词
- 同义词
- 平台适配词
- 排除词
- 竞品词
- 人群词
- 场景词

### 4.5 采集计划

展示计划和执行情况：

- 计划名称
- 平台
- 采集对象
- 目标样本
- 当前进度
- 状态
- 下一次运行时间
- 失败原因

支持动作：

- 立即运行
- 暂停
- 停止当前运行
- 重试失败项
- 编辑采集策略
- 生成补采计划

采集单位明细：

- 每个平台
- 每个关键词
- 每个达人
- 每个内容 ID

### 4.6 样本与证据

这是数据浏览页的业务化替代。

筛选：

- 平台
- 关键词
- 达人
- 内容类型
- 时间范围
- 互动区间
- 来源任务
- 是否被结论引用

样本列表展示：

- 标题/正文摘要
- 平台
- 作者
- 发布时间
- 互动指标
- 命中关键词
- 标签
- 关联结论

点击打开内容详情抽屉：

- 原文
- 链接
- 互动指标
- 评论摘要
- 命中词
- 标签
- 来源任务
- 支撑了哪些判断

### 4.7 洞察与建议

展示项目级 AI/规则结论：

- 本项目当前最大的机会。
- 当前样本是否足够。
- 哪些关键词值得继续。
- 哪些达人值得跟进。
- 哪些友商需要观察。
- 哪些内容模式值得复用。

每条洞察必须有：

- 结论
- 证据
- 风险
- 建议动作

### 4.8 历史任务

展示当前项目的任务，不展示全局无关任务。

任务类型：

- 关键词采集
- 达人发现
- 内容追踪
- 友商监控
- AI 分析
- 补采任务

失败任务展示可读诊断。

### 4.9 导出

项目内导出：

- 完整项目报告
- 达人名单
- 内容样本
- 关键词热度
- 友商日报
- 证据包

格式：

- Excel
- CSV
- Markdown
- JSON

## 5. 页面三：达人发现

### 页面定位

达人发现页用于从业务描述或关键词中发现可跟进达人，并完成筛选、分层、证据审查和后续动作。

### 核心用户路径

1. 用户输入人群或场景描述。
2. 系统生成关键词和标签。
3. 系统从已有样本中召回达人。
4. 样本不足时建议补采。
5. 系统补全达人主页和近期内容。
6. 输出 A/B/C 分层结果。
7. 用户加入候选池、友商池或导出。

### 页面布局

顶部：

- 当前项目。
- 输入框：描述目标达人。
- 平台选择。
- 高级筛选。
- 开始发现按钮。

中部：

- 发现进度。
- 样本质量。
- 达人分层结果。

右侧或抽屉：

- 达人详情。
- 匹配证据。
- 操作记录。

### 输入区

支持三种输入：

- 自然语言描述。
- 关键词组合。
- 从项目配置导入。

示例：

```text
找宠物主粮领域，粉丝 1 万到 20 万，近 30 天持续更新，内容以真实测评和喂养经验为主的达人。
```

高级筛选：

- 平台
- 粉丝范围
- 近 30 天发文数
- 最低互动率
- 是否排除机构号
- 是否排除疑似友商
- 是否要求有联系方式线索

### 达人发现流程

系统内部流程应拆成可见步骤：

1. 理解人群描述。
2. 生成关键词和标签。
3. 检索本地样本。
4. 补采缺口样本。
5. 反推作者。
6. 补全达人主页。
7. 计算评分。
8. 生成分层和证据。

每一步显示状态：

- 待执行
- 运行中
- 完成
- 失败
- 已跳过

失败时给出原因和操作：

- 关键词太窄，建议扩词。
- 平台无数据，建议换平台。
- API 不可用，去设置。
- 样本不足，建议补采。

### 达人列表

列表支持三种视图：

- 分层视图：A/B/C 档。
- 表格视图：适合批量筛选。
- 卡片视图：适合运营浏览。

每个达人展示：

- 昵称
- 平台
- 主页链接
- 粉丝数
- 近 30 天发文数
- 平均互动率
- 爆款率
- 匹配分
- 商业化信号
- 风险标签

分层含义：

- A 档：建议立即跟进。
- B 档：建议观察或补证据。
- C 档：低优先级。
- 疑似友商：可加入友商监控。
- 样本不足：需要补采后判断。

### 达人评分卡

点击达人打开详情抽屉，包含：

- 基础信息
- 匹配解释
- 代表内容
- 近期趋势
- 标签画像
- 商业化信号
- 风险提示
- 后续动作

评分维度：

- 垂直相关度
- 内容活跃度
- 互动质量
- 爆款能力
- 人群匹配度
- 商业化风险
- 数据可信度

每个维度显示分数和证据。

### 代表内容证据

展示 3 到 10 条最能解释匹配的内容：

- 标题
- 摘要
- 发布时间
- 互动指标
- 命中关键词
- 命中标签
- 链接

交互：

- 查看原文
- 加入内容追踪
- 标记为强证据
- 标记为无关

### 批量操作

支持：

- 加入候选池
- 加入友商池
- 创建监控
- 导出名单
- 标记不相关
- 批量补全主页

### 当前已有功能

- 关键词筛选达人。
- 候选达人池。
- 加入候选池。
- 补全主页指标。
- 平台分布。

### 需要优化

- 从“筛本地数据”升级为“发现任务”。
- 增加分层和评分解释。
- 增加样本不足诊断。
- 强化代表内容证据。

### 需要新增

- 自然语言达人发现入口。
- AI 生成关键词和标签。
- 自动补采。
- 达人评分卡。
- A/B/C 分层。
- 友商风险识别。

## 6. 页面四：内容追踪

### 页面定位

内容追踪页用于从一条内容、一段文案或一个链接出发，定位关键词、找到同类内容、拆解内容模式，并持续追踪同类内容变化。

### 核心用户路径

1. 用户输入内容。
2. 系统解析关键词和内容特征。
3. 系统搜索同类内容。
4. 系统拆解内容模式和表现。
5. 用户创建追踪器。
6. 系统持续监控新增同类内容和互动变化。

### 页面布局

顶部输入区：

- 粘贴文本
- 粘贴链接
- 上传字幕文本
- 选择已有样本

中部：

- 内容解析结果
- 同类内容列表
- 内容模式拆解

右侧：

- 追踪器配置
- 样本质量
- 操作建议

### 内容输入

支持：

- 标题和正文。
- 短视频描述。
- 评论区片段。
- 平台内容链接。
- 已采集内容样本。
- 视频字幕文本。

首版可先不做完整视频画面理解，但界面要预留：

- 字幕/OCR
- 画面标签
- 语音转写

### 内容解析

解析输出：

- 核心关键词
- 扩展关键词
- 排除词
- 人群词
- 场景词
- 痛点词
- 卖点词
- 情绪词
- 内容类型
- 目标受众
- 可能的转化意图

每个词显示：

- 类型
- 置信度
- 出现位置
- 是否加入追踪

交互：

- 勾选加入追踪器。
- 编辑关键词。
- 添加排除词。
- 重新解析。

### 同类内容搜索

搜索逻辑：

- 关键词召回。
- 标签匹配。
- 作者/达人关联。
- 语义相似。
- 互动表现筛选。

首版可以先做关键词召回 + 标签匹配 + AI 解释，后续加入向量检索。

列表字段：

- 标题
- 平台
- 作者
- 发布时间
- 互动指标
- 相似度
- 相似原因
- 命中关键词

筛选：

- 平台
- 时间
- 互动区间
- 内容类型
- 相似原因
- 是否爆款

### 内容模式拆解

## 13. 首版验收标准

业务可用首版应满足：

- 用户能通过向导创建任意行业项目。
- 用户能从一个项目进入四个工作流。
- 每个工作流都有明确输入、结果、证据和下一步动作。
- 达人发现能输出分层名单和匹配证据。
- 内容追踪能从一段内容生成关键词和同类内容。
- 友商监控能输出每日流量增量和贡献内容。
- 关键词热度能输出状态判断和样本质量。
- 所有失败状态都能给出可执行处理建议。
- 所有结论都能点击查看证据。

## 14. 不做或后置

首版不建议做：

- 复杂大屏。
- 完整权限系统。
- 过度自动化的 AI Agent。
- 没有证据的长篇策略报告。
- 强依赖视频画面理解的结论。
- 复杂机器学习模型。

首版应优先保证业务动作闭环，而不是追求算法复杂度。

## 15. 用户角色与权限边界

首版可以不做复杂权限系统，但必须明确主要用户角色，否则页面会同时服务老板、运营、分析和管理员，导致交互失焦。

### 15.1 老板/增长负责人

- 快速知道今天是否有机会。

- 看项目是否健康。

- 今日情报
- 项目概览

- 标记机会有效/观察/误判

不建议暴露：

- 原始 raw records
- 复杂采集参数
- API key 配置
- 任务队列细节

### 15.2 内容运营/增长运营

核心目标：

- 创建项目。
- 找达人。
- 追踪同类内容。

## 16. 关键指标口径

页面上所有分数都必须有口径说明，至少在悬浮提示或详情抽屉中解释。首版可以用规则评分，但不能让用户误以为这是不可解释的黑盒模型。

### 16.1 样本质量分

用途：

- 判断某个结论是否可用于业务决策。
- 适用于达人、内容、友商、关键词、项目整体。

输入指标：

- 近 24 小时内容数。
- 近 7 天内容数。
- 近 7 天参与达人数。
- 平台覆盖数。
- 采集成功率。
- 是否有历史基线。
- 数据新鲜度。

等级：

- 高：可以直接用于决策。
- 中：可以作为运营线索，需要人工复核。
- 低：只能观察，不能作为强结论。
- 不足：必须补采后再判断。

建议首版阈值：

- 高：7 天内容不少于 100 条，达人不少于 20 个，采集成功率不低于 80%。
- 中：7 天内容不少于 40 条，达人不少于 8 个，采集成功率不低于 60%。
- 低：有样本但低于中等级。
- 不足：无样本、只有单平台极少样本，或最近采集失败。

### 16.2 达人匹配分

用途：

- 对候选达人排序和分层。

输入指标：

- 关键词命中。
- 标签命中。
- 近 30 天相关内容占比。
- 近 30 天发文频率。
- 平均互动率。
- 爆款率。
- 粉丝规模是否符合筛选。
- 商业化/友商风险。
- 样本质量。

建议权重：

- 垂直相关度：35%
- 活跃度：15%
- 互动质量：20%
- 爆款能力：15%
- 人群匹配：10%
- 风险扣分：最多扣 20 分
- 样本质量修正：低质量样本最高只能给 B 档

分层规则：

- A 档：匹配分不低于 80，样本质量中或高。
- B 档：匹配分 60 到 79，或样本质量偏低但信号明显。
- C 档：匹配分低于 60。
- 观察：数据不足但有明显命中。
- 疑似友商：商业化/品牌/机构信号明显。

### 16.3 内容相似度

用途：

- 判断同类内容是否值得追踪或复用。

输入指标：

- 关键词重合。
- 标签重合。
- 目标人群一致。
- 场景一致。
- 痛点一致。
- 标题/结构相似。
- 评论反馈相似。
- 互动表现相近或更优。

首版策略：

- 关键词和标签召回为主。
- AI 解释相似原因。
- 后续加入向量检索。

结果必须展示“相似原因”，例如：

- 都命中同一核心词。
- 都面向同一人群。
- 都围绕同一痛点。
- 都使用相似内容结构。

### 16.4 友商每日流量增量

用途：

- 判断友商今天的公开流量变化。

口径：

- 以内容公开互动指标为代理。
- 总互动 = 点赞 + 评论 + 收藏 + 分享。
- 今日增量 = 今日快照总互动 - 上一次快照总互动。
- 新内容贡献 = 今日新发布内容的互动增量。
- 老内容贡献 = 历史内容在今日产生的互动增量。

必须展示：

- 当前快照时间。
- 上一次快照时间。
- 使用的内容范围。
- 是否全量或最新 N 条。
- 本次采集是否完整。

首版注意：

- 如果平台只能拿最新 N 条，需要明确标注“非全量”。
- 如果没有上一日快照，只能展示累计值，不能展示真实增量。

### 16.5 关键词热度分

用途：

- 判断关键词是否值得测试、观察、补采或暂缓。

输入指标：

- 24 小时内容量。
- 7 天平均内容量。
- 30 天平均内容量。
- 24 小时互动量。
- 爆款率。
- 参与达人数。
- 新达人参与数。
- 友商是否参与。
- 样本质量。

状态：

- 推流中：内容量和互动量同时明显高于基线，且爆款率不低。
- 正常：内容量和互动量接近基线。
- 降温中：内容量或互动量低于基线。
- 疑似限流：内容量存在但互动显著低，或同类内容持续低互动。
- 样本不足：样本质量低或无历史基线。

首版必须避免的问题：

- 不能因为没有数据就判断限流。
- 不能因为单条爆款就判断平台推流。
- 不能跨平台混用同一阈值下强结论。

### 16.6 机会分

用途：

- 在今日情报中排序机会。

输入指标：

- 热度或增长信号。
- 样本质量。
- 可执行性。
- 竞争强度。
- 风险。
- 与项目目标匹配度。

建议规则：

- 机会分 = 增长信号 35% + 样本质量 25% + 可执行性 20% + 项目匹配 20% - 风险扣分。

机会卡必须展示：

- 分数。
- 分数构成。
- 证据数量。
- 风险。
- 建议动作。

## 17. 四个工作流的联动闭环

四个工作台不能孤立存在。用户在任何一个页面发现信号，都应该能继续推进到下一个动作。

### 17.1 关键词到内容

触发条件：

- 关键词状态为推流中。
- 关键词样本质量中或高。
- 关键词相关内容出现增长。

动作：

- 一键创建内容追踪器。
- 自动带入核心词、扩展词、排除词。
- 自动推荐代表内容作为追踪种子。

结果：

- 内容追踪页持续监控同类内容。
- 今日情报展示新增内容机会。

### 17.2 内容到达人

触发条件：

- 内容追踪发现高互动同类内容。
- 某类内容模式持续出现。

动作：

- 从代表内容反推作者。
- 创建达人发现任务。
- 按内容模式筛选达人。

结果：

- 达人发现页输出相关达人。
- 可加入候选池或友商池。

### 17.3 达人到友商

触发条件：

- 达人有明显商业化信号。
- 达人内容长期围绕产品/服务。
- 达人疑似机构或品牌账号。

动作：

- 加入友商池。
- 创建每日监控。
- 首次回溯该账号内容。

结果：

- 友商监控页开始生成每日流量账本。

### 17.4 友商到关键词

触发条件：

- 友商某条内容突然增长。
- 友商内容主题发生变化。
- 友商持续押注某个关键词。

动作：

- 从友商内容反推关键词。
- 创建关键词热度分析。
- 创建同类内容追踪。

结果：

- 关键词热度页判断该词是否值得跟。
- 内容追踪页追踪同类内容。

### 17.5 任何结论到今日情报

所有工作流产出的强信号都进入今日情报：

- A 档达人。
- 推流关键词。
- 友商异常。
- 新内容模式。
- 样本不足但潜力高的观察项。

今日情报负责统一排序和行动建议。

## 18. 页面状态规范

采集和分析类产品不能只设计成功态。下面状态必须在所有工作台中统一处理。

### 18.1 加载中

场景：

- 页面首次打开。
- 切换项目。
- 切换筛选条件。

展示：

- 骨架屏或轻量 loading。
- 保留上次结果，避免页面闪烁。

### 18.2 排队中

场景：

- 任务已创建但未开始。

展示：

- 当前排队位置。
- 预计开始时间。
- 取消按钮。

### 18.3 采集中

场景：

- 采集子任务运行中。

展示：

- 平台。
- 当前采集对象。
- 已采样本。
- 目标样本。
- 已耗时。
- 最近日志摘要。

### 18.4 分析中

场景：

- 采集完成后正在打标签、评分、AI 分析或生成报告。

展示：

- 当前分析阶段。
- 已完成项。
- 预计剩余。

### 18.5 部分成功

场景：

- 多平台中部分平台成功。
- 部分关键词有结果。
- 部分任务失败但已有可用样本。

展示：

- 成功部分。
- 失败部分。
- 当前结果是否可用。
- 是否建议补采。

### 18.6 样本不足

场景：

- 采集成功但样本达不到判断门槛。

展示：

- 当前样本量。
- 目标样本量。
- 缺口。
- 建议扩展词或平台。
- 一键补采。

### 18.7 结果过期

场景：

- 上次采集时间超过项目设定窗口。
- 友商快照不是今天。
- 关键词热度超过 24 小时未更新。

展示：

- 上次更新时间。
- 过期原因。
- 重新采集按钮。

### 18.8 配置缺失

场景：

- 缺 API key。
- AI Provider 未配置。
- 平台能力被禁用。

展示：

- 缺什么。
- 影响哪些功能。
- 去设置按钮。

### 18.9 采集失败

场景：

- 爬虫退出。
- API 认证失败。
- 平台限制。
- 入库失败。

展示：

- 用户可读原因。
- 原始错误摘要。
- 推荐处理方式。
- 重试按钮。

错误原因必须归类：

- 凭证问题
- 平台问题
- 关键词问题
- 账号问题
- 网络/API 问题
- 入库问题
- 后处理问题

## 19. P0 首版范围硬边界

为了保证业务可用首版能落地，P0 必须收敛。

### 19.1 今日情报 P0

必须做：

- 今日行动清单。
- 机会队列。
- 风险与样本质量。
- 机会详情抽屉。

可以简化：

- 趋势图只做基础折线或数字变化。
- 今日摘要可以先用规则模板，AI 不可用时降级。

不做：

- 复杂大屏。
- 多人协作指派。
- 自动推送。

### 19.2 项目工作台 P0

必须做：

- 项目创建向导。
- AI/规则生成研究配置。
- 采集计划预览。
- 项目样本与证据。
- 项目历史任务。

可以简化：

- 标签和权重先使用默认模板。
- 导出先支持 CSV/Excel 之一。

不做：

- 复杂模板市场。
- 多版本配置管理。

### 19.3 达人发现 P0

必须做：

- 自然语言输入。
- 关键词/标签生成。
- 本地召回 + 必要补采。
- 达人列表。
- A/B/C 分层。
- 达人详情和证据。
- 加入候选池/友商池。

可以简化：

- 粉丝和互动字段以平台可获取为准。
- 商业化风险先用规则识别。

不做：

- 私信线索自动获取。
- 自动联系达人。

### 19.4 内容追踪 P0

必须做：

- 文本/链接/已有样本输入。
- 关键词定位。
- 同类内容列表。
- 内容模式摘要。
- 创建追踪器。

可以简化：

- 视频先支持标题、简介、字幕文本。
- 同类内容先用关键词和标签召回。

不做：

- 完整视频画面理解。
- 向量数据库。

### 19.5 友商监控 P0

必须做：

- 新增友商。
- 首次回溯。
- 每日增量快照。
- 今日流量账本。
- 内容贡献排行。
- 基础组成模式。
- 异常提示。

可以简化：

- 如果平台只能拿最新 N 条，明确标记非全量。
- 组成模式先基于关键词、标签和发布时间。

不做：

- 预测友商下一步动作。
- 复杂归因模型。

### 19.6 关键词热度 P0

必须做：

- 关键词雷达。
- 关键词详情。
- 热度状态。
- 样本质量。
- 补采建议。
- 证据内容。

可以简化：

- 先使用规则评分。
- 平台权重先配置化，不做自动学习。

不做：

- 无证据的强推流结论。
- 跨平台统一黑盒评分。

## 21. 爬虫与采集系统设计补充

### 21.1 平台能力矩阵

设置页需要展示每个平台能力，后台也要按能力决定是否允许创建任务。

能力项：

- keyword_search
- content_detail
- comments
- sub_comments
- creator_profile
- creator_posts
- engagement_metrics
- publish_time
- search_sort
- time_filter

每项状态：

- supported
- unsupported
- disabled
- requires_key
- unstable

### 21.2 采集模式

统一支持：

- keyword_search：按关键词采内容。
- content_detail：按内容 ID 补详情。
- content_comments：按内容采评论。
- creator_profile：补达人主页。
- creator_posts：采达人作品列表。
- competitor_monitor：友商每日监控。
- similar_content_search：同类内容搜索。

### 21.3 首次回溯与增量采集

首次回溯：

- 项目创建后按关键词、达人、友商进行基础采集。
- 用于建立初始样本和基线。

增量采集：

- 每日只采新增内容和指标变化。
- 对友商和内容追踪器必须保存快照。
- 对关键词热度必须保存历史窗口。

### 21.4 重试策略

错误分为可重试和不可重试。

可重试：

- 网络超时。
- API 5xx。
- 平台临时限流。
- 入库短暂失败。

不可自动重试：

- API key 缺失。
- API key 无效。
- 平台能力不支持。
- 账号 ID 明显无效。
- 关键词为空。

重试策略：

- 默认最多 3 次。
- 指数退避。
- 多次失败后生成诊断建议。

### 21.5 采集质量诊断

每次任务结束后生成诊断：

- started_count
- success_count
- failed_count
- empty_count
- output_posts
- output_comments
- output_creators
- error_breakdown
- sample_quality
- suggested_next_action

诊断示例：

```text
本次抖音关键词采集成功执行 8 个关键词，但 5 个关键词无结果。建议扩展关键词或降低时间过滤条件。
```

### 21.6 入库与后处理校验

采集成功不等于业务可用。必须校验：

- raw 是否写入。
- 标准内容表是否写入。
- 作者是否关联。
- 指标是否解析。
- 标签是否生成。
- 证据是否创建。

如果采集成功但后处理失败，页面应显示“采集成功，分析未完成”，并允许重跑后处理。

## 22. AI 使用边界

### 22.1 AI 可以做什么

首版允许 AI 做：

- 根据业务语境生成关键词。
- 生成排除词。
- 生成标签定义。
- 解释达人匹配原因。
- 拆解内容模式。
- 生成友商变化摘要。
- 辅助解释关键词热度。
- 生成日报草稿。

### 22.2 AI 不直接做什么

AI 不应直接决定：

- 关键词是否一定被平台推流。
- 某达人是否一定值得投放。
- 某友商是否一定投流。
- 样本不足时的强结论。

这些必须结合规则、样本质量和证据。

### 22.3 AI 输出格式

AI 输出必须结构化：

- conclusion
- evidence_refs
- risk_notes
- suggested_actions
- confidence
- assumptions

禁止只保存长文本。

### 22.4 降级策略

AI 不可用时：

- 项目配置使用规则模板。
- 达人匹配使用关键词和标签规则。
- 友商日报使用指标模板。
- 关键词热度使用规则评分。

页面应标注：

```text
AI 暂不可用，当前为规则分析结果。
```

### 22.5 可追溯性

## 24. 业务验收用例

### 用例一：创建通用行业项目

输入：

```text
我想研究宠物主粮在小红书和抖音上的内容机会，目标是找达人、看关键词热度、监控竞品。
```

期望：

- 系统生成项目配置。
- 生成关键词、排除词、标签。
- 展示采集计划。
- 用户确认后启动采集。
- 项目概览显示采集进度和样本质量。

### 用例二：根据关键词发现达人

输入：

```text
宠物主粮 + 新手养猫
```

期望：

- 输出达人列表。
- 至少有 A/B/C 分层。
- 每个达人有匹配原因和代表内容。
- 样本不足时提示补采建议。
- 用户可以加入候选池或导出。

### 用例三：根据内容追踪同类内容

输入：

- 一段平台文案或一条已采集内容。

期望：

- 系统提取核心关键词和排除词。
- 展示同类内容列表。
- 说明相似原因。
- 输出内容模式摘要。
- 用户可创建追踪器。

### 用例四：每日友商监控

输入：

- 添加一个小红书或抖音友商账号。

期望：

- 系统完成首次回溯。
- 次日生成每日流量账本。
- 展示新老内容贡献。
- 展示贡献内容排行。
- 出现异常时有解释和证据。

### 用例五：关键词热度判断

输入：

```text
新手养猫主粮
```

期望：

- 展示关键词状态。
- 展示 24h/7d/30d 指标。
- 展示样本质量。
- 判断推流、正常、降温、疑似限流或样本不足。
- 给出补采或运营动作建议。

### 用例六：失败诊断

输入：

- 关闭或删除 TikHub API key 后运行采集。

期望：

- 前端不只显示退出码。
- 显示“缺少 API key”。
- 显示影响的功能。
- 提供去设置入口。
