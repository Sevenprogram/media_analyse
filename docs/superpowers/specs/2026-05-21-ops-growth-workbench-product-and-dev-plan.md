# 运营增长工作台产品设计与开发计划

日期：2026-05-21

## 1. 背景与目标

当前系统已经具备采集、达人搜索、内容追踪、友商监控、关键词热度和运营总览的基础能力，但这些能力仍偏分散。第一阶段目标不是做全平台全量监控，也不是判断平台算法真相，而是搭建一条稳定的运营增长情报闭环：

```text
人工主词
-> AI 扩展词包
-> 人工确认
-> 每日关键词采样
-> 文本内容分析
-> 达人画像与 A/B/C 分层
-> A 档自动进入监控池
-> 达人/友商定时监控
-> 生成内容、达人、友商、关键词热度证据
-> 运营工作台持续筛选、纠错、跟进
```

第一阶段重点平台：

- 小红书优先。
- 抖音第二。

第一阶段分析范围：

- 只分析公开文本数据，包括标题、正文、话题、简介、评论文本、公开互动指标。
- 暂不做视频 ASR、OCR、画面识别、镜头结构分析。
- 不承诺“全部流量”，只基于已采集公开样本和监控账号公开内容做分析。

## 2. 产品定位

产品定位为“运营增长工作台”，服务对象是运营人员，而不是老板大屏。

运营人员需要完成的核心动作：

- 配置追踪词包。
- 查看每日采集内容。
- 追踪同类内容。
- 查看达人 A/B/C 候选池。
- 审核和管理自动入池记录。
- 管理达人监控池。
- 管理友商账号。
- 查看友商内容和流量组成。
- 查看关键词热度、推流/降温线索和样本置信度。
- 手动纠错标签、达人档位、友商身份。

当前总览页暂不优化，第一阶段重点是把已有页面和后端能力打通为可用工作流。

## 3. 第一阶段范围

### 3.1 已确认策略

- 平台：小红书优先，抖音第二。
- 采集模式：混合模式。
  - 每天先跑关键词采样。
  - 从采样内容中发现达人。
  - 高分达人进入候选池。
  - A 档达人自动进入监控池。
  - 已确认友商进入重点监控。
- AI 使用强度：中 AI。
  - AI 做词包扩展、达人标签建议、内容摘要、异常原因解释。
  - 规则做评分、分层、自动入池、热度判断、样本置信度。
- 历史窗口：30 天。
  - 24h 看当天异常。
  - 7d 看短期趋势。
  - 30d 做基线和复盘。
- 采集规模：
  - 每个追踪词每天 100-200 条。
  - 每个词包通常 5-15 个有效搜索词。
  - 每日目标样本量为几千条。
- 监控频率：
  - A 档达人每天 2 次。
  - B 档观察池每天 1 次。
  - 重点友商每天 3 次。
  - C 档只保留证据，不主动监控。
- A/B/C 分层：
  - A 档：score >= 75，自动进入监控池。
  - B 档：60 <= score < 75，观察。
  - C 档：score < 60，只保留证据。
- 友商来源：
  - 人工输入账号 ID 或主页链接。
  - 关键词发现疑似友商。
  - 从 A 档达人中推荐疑似商业账号/竞品账号。

### 3.2 暂不做

- 不新增老板 dashboard。
- 不优化当前总览页。
- 不做视频理解。
- 不做全网全量覆盖承诺。
- 不做外部通知，如飞书、企微、邮件。
- 不做完全依赖 AI 的黑盒评分。

## 4. 核心用户流程

### 4.1 词包追踪流程

```text
运营输入主词
-> AI 生成扩展词、同义词、平台适配词、排除词
-> 运营确认词包
-> 系统每天按词包采集内容
-> 内容入库并记录来源词、命中词、平台、采集批次
-> 生成内容追踪快照
-> 计算关键词热度和置信度
```

词包结构建议：

```json
{
  "name": "K12教育 + 单亲妈妈",
  "platforms": ["xhs", "dy"],
  "seed_keywords": ["K12教育", "单亲妈妈"],
  "positive_keywords": ["陪读妈妈", "小学提分", "英语启蒙"],
  "platform_keywords": {
    "xhs": ["小红书适配词"],
    "dy": ["抖音适配词"]
  },
  "negative_keywords": ["情感八卦", "无关鸡汤"],
  "daily_sample_limit_per_keyword": 150,
  "enabled": true
}
```

### 4.2 内容追踪流程

```text
采集内容
-> 文本清洗
-> 关键词命中
-> 标签命中
-> AI 摘要和内容指纹
-> 同类内容聚合
-> 输出高互动内容、内容模式、趋势变化
```

内容指纹字段建议：

```json
{
  "audience": "小学家长",
  "pain_point": "孩子英语基础差",
  "topic": "K12英语启蒙",
  "content_type": "经验分享",
  "conversion_intent": "课程咨询",
  "summary": "单亲妈妈分享孩子英语启蒙经验"
}
```

第一阶段同类内容排序不必直接上复杂向量库，可以先用：

- 关键词命中。
- 标签命中。
- 内容指纹相似。
- 互动表现加权。

后续再升级为 embedding 语义检索。

### 4.3 达人发现流程

```text
从采集内容提取作者
-> 聚合作者近 30 天内容
-> 规则打标签
-> AI 受控补标签
-> 生成达人画像
-> 计算候选分
-> 分为 A/B/C 档
-> A 档自动入监控池
-> B 档观察
-> C 档只保留证据
```

达人候选分建议：

```text
达人候选分 =
关键词/标签匹配 40%
近 30 天相关内容数量 25%
单篇内容互动表现 20%
账号基础质量 10%
证据完整度 5%
负面词/低质信号为惩罚项
```

AI 补标签必须受控：

- 只能从已启用标签库选择标签。
- 必须给置信度。
- 必须给证据。
- 不允许直接生成未配置标签并用于评分。
- AI 提议的新标签只能进入“待确认标签建议”。

### 4.4 自动入池流程

```text
候选达人分层完成
-> A 档 score >= 75
-> 检查自动入池保护条件
-> 未超过每日自动入池上限
-> 加入监控池
-> 创建或更新 creator monitor job
-> 记录自动入池来源和原因
```

保护条件：

- 近 30 天至少有相关内容。
- 至少命中 2 类证据：标签、关键词、内容数量、互动表现、账号简介。
- 命中负面词不得自动入池。
- 每日自动新增监控达人默认上限 20 个。
- 连续 7 天无相关内容时降级或暂停监控。

### 4.5 友商监控流程

```text
人工录入友商账号 ID / 主页链接
或系统推荐疑似友商
-> 创建友商账号
-> 每天 3 次采集公开内容和互动指标
-> 生成友商快照
-> 生成流量组成快照
-> 展示关键词、内容类型、发布时间、互动结构、爆款率
```

疑似友商判断信号：

- 简介或内容出现课程、咨询、训练营、资料包、私域等词。
- 高频发布同一赛道内容。
- 内容存在明显转化意图。
- 互动表现稳定。
- 账号体量或增长速度达到阈值。

### 4.6 关键词热度判断流程

```text
按词包统计 24h、7d、30d 样本
-> 对比内容量、互动量、爆款数、参与达人数
-> 计算热度分、推流线索、降温线索
-> 输出置信度
-> 样本不足时输出补采建议
```

输出不应只给一个标签，而应包含：

- 当前判断：疑似升温、疑似降温、正常波动、样本不足。
- 热度分。
- 推流线索分。
- 降温风险分。
- 置信度。
- 样本量。
- 证据。
- 补采建议。

样本不足示例：

```text
疑似升温，置信度 42%。
原因：近 7 天样本 18 条，达人覆盖 5 个，低于可靠判断阈值。
建议：继续采集“陪读妈妈、小学提分、英语启蒙”等扩展词，目标 7 天样本达到 80 条后再判断。
```

### 5.1 已有能力

- 采集任务与调度：
  - `research/scheduler.py`
  - `research/execution.py`
  - `api/routers/research.py`

  - `research/tagging.py`

  - `research/monitor_pools.py`

  - `research/content_tracking.py`

  - `research/competitors.py`
  - `api/routers/competitors.py`

  - `research/keyword_library.py`

  - `api/webui/src/pages/ResearchModulePages.tsx`

### 5.2 主要缺口

- 词包还没有完整表达追踪策略。
- AI 扩词缺少人工确认后生效闭环。
- 每日关键词采样还没有按词包自动拆任务。
- 采集结果缺少明确的词包归因。
- 内容分析缺少摘要、指纹、结构化分类。
- 达人候选池缺少 A/B/C 档位和状态机。
- A 档自动入监控池还没有完整自动化。
- 监控频率还没有按 A/B/友商分层。
- 友商采集完成后自动生成组成快照还不顺滑。
- 关键词热度缺少动态基线和补采建议。
- 运营页面缺少自动入池记录、档位筛选、样本置信度解释等操作视图。

## 7. 后端服务设计

### 7.1 词包服务

职责：

- 创建和维护追踪词包。
- 调用 AI 生成扩展词。
- 保存 AI 建议。
- 人工确认后启用词包。
- 提供待调度词包列表。

建议模块：

- `research/tracking_packs.py`
- `api/routers/tracking_packs.py`

### 7.2 采样调度服务

职责：

- 每天按启用词包拆分采集任务。
- 为每个平台、每个关键词生成搜索任务。
- 控制每个词每日采集量。
- 记录调度批次。

建议模块：

- `research/tracking_scheduler.py`

### 7.3 采集后分析服务

职责：

- 采集完成后自动运行。
- 内容打标。
- 内容摘要和指纹。
- 达人画像重建。
- 候选达人分层。
- 自动入池。
- 内容追踪快照。
- 关键词热度快照。

建议模块：

- 增强 `research/postprocess.py`
- 新增 `research/candidate_tiering.py`
- 新增 `research/content_fingerprint.py`

### 7.4 自动入池服务

职责：

- 根据 A/B/C 分层决定是否入池。
- 检查每日上限。
- 检查负面条件。
- 创建或更新监控池任务。

建议模块：

### 7.5 友商快照服务

建议模块：

- 增强 `research/competitors.py`

### 7.6 热度与置信度服务

职责：

- 计算 24h/7d/30d 指标。

- 增强 `research/keyword_heat.py`

## 9. 开发计划

### 阶段 0：整理现有能力和数据迁移

目标：

- 确认当前数据库表、API 和页面状态。
- 不改变业务行为，先补齐模型兼容和测试基线。

任务：

- 梳理现有 keyword library、scene pack、content tracker、creator candidate、monitor pool、competitor snapshot 表。
- 确认哪些字段可复用，哪些字段必须新增。
- 为新增字段写 schema migration。
- 为核心 API 加回归测试。

验收：

- 现有测试通过。
- 前端 build 通过。
- 当前总览页不受影响。

### 阶段 1：追踪词包与 AI 扩词确认

目标：

- 运营可以创建词包、AI 扩展、人工确认启用。

后端任务：

- 增强关键词库或新增 tracking pack 模型。
- 增加词包 CRUD API。
- 增加 AI 扩词建议 API。
- 增加确认启用 API。

前端任务：

- 词包表单。
- AI 扩词结果预览。
- 正向词、平台词、排除词编辑。
- 启用状态和最近采样状态展示。

验收：

- 输入 `K12教育 + 单亲妈妈` 后能生成扩展词。
- 人工确认后词包进入启用状态。
- 未确认词包不会进入每日采样。

### 阶段 2：词包每日采样任务

目标：

- 系统能按启用词包自动生成小红书关键词采样任务，后续支持抖音。

后端任务：

- 新增词包调度服务。
- 按平台和关键词拆分 research job 或 crawl unit。
- 支持每个词每天 100-200 条配置。
- 记录 source keyword、scene pack、采集批次。

前端任务：

- 展示词包最近采集任务。
- 展示每个关键词采集数量、成功/失败状态。

验收：

- 启用词包后能创建小红书采样任务。
- 同一内容重复命中多个词时能够归因和去重。
- 采集失败能在工作台看到原因。

### 阶段 3：采集后文本分析

目标：

- 采集完成后自动生成内容标签、摘要和内容指纹。

后端任务：

- 增强 postprocess 流程。
- 规则关键词命中。
- AI 摘要和内容指纹。
- 保存分析结果。
- 生成内容追踪快照。

前端任务：

- 内容追踪页展示摘要、内容类型、人群、痛点、转化意图。
- 支持按标签和关键词筛选内容。

验收：

- 每次采集后自动生成内容分析结果。
- 运营能看到同类内容列表和证据。
- 不做视频理解也能基于文本完成分析。

### 阶段 4：达人画像、A/B/C 分层和自动入池

目标：

- 从采集内容中沉淀达人，自动分层，A 档自动入监控池。

后端任务：

- 重建达人画像。
- 计算候选分。
- 保存 A/B/C 档位。
- 实现 A 档自动入池。
- 实现每日自动入池上限。
- 实现负面词拦截。

前端任务：

- 候选池支持 A/B/C 筛选。
- 展示候选分、原因、证据。
- 展示自动入池记录。
- 支持手动调整档位和入池状态。

验收：

- score >= 75 的达人自动入池。
- B 档进入观察。
- C 档只保留证据。
- 自动入池有日志和原因。

### 阶段 5：监控池分层调度

目标：

- 不同类型对象使用不同监控频率。

后端任务：

- A 档达人每天 2 次。
- B 档每天 1 次。
- 重点友商每天 3 次。
- 连续 7 天无相关内容降级或暂停。

前端任务：

- 监控池展示监控频率。
- 展示最近采集时间。
- 展示无相关内容天数。
- 支持暂停/恢复。

验收：

- A/B/友商调度频率不同。
- 自动降级/暂停有原因记录。
- 运营可以手动恢复。

### 阶段 6：友商多入口和组成快照自动化

目标：

- 友商可以人工添加，也可以系统推荐；采集完成后自动生成流量组成。

后端任务：

- 支持账号 ID 和主页链接添加友商。
- 从 A 档达人中识别疑似友商。
- 友商采集后自动生成 daily snapshot。
- 自动生成 composition snapshot。

前端任务：

- 友商添加表单。
- 疑似友商推荐列表。
- 友商组成视图。

验收：

- 手动添加友商后能进入重点监控。
- 友商内容采集后自动出现组成拆解。
- 能看到关键词、内容类型、发布时间、互动结构、爆款率。

### 阶段 7：关键词热度、动态基线和补采建议

目标：

- 关键词判断更可解释，样本不足时知道下一步怎么补。

后端任务：

- 增强 24h/7d/30d 计算。
- 输出样本置信度。
- 输出补采建议。
- 按平台、词包、内容类型逐步增加基线。

前端任务：

- 展示热度分、推流线索、降温风险。
- 展示置信度和样本原因。
- 展示补采建议。

验收：

- 样本不足不输出确定结论。
- 系统给出具体补采建议。
- 运营能看懂为什么判断升温或降温。

## 11. 风险与控制

### 11.1 采集稳定性风险

风险：

- 每日几千条样本可能触发平台风控或失败率升高。

控制：

- 每词采集量可配置。
- 平台级每日上限。
- 失败重试次数上限。
- 失败原因写入工作台。

### 11.2 AI 成本和不稳定风险

风险：

- AI 摘要、标签、扩词会产生成本。
- AI 可能生成不稳定结果。

控制：

- AI 只做理解和建议。
- 决策由规则完成。
- AI 标签必须从标签库选择。
- AI 新标签必须人工确认。
- 对同一内容分析结果做缓存。

### 11.3 自动入池误判风险

风险：

- 激进入池可能把弱相关达人加入监控。

控制：

- 每日自动入池上限。
- 负面词拦截。
- 自动入池证据必须可见。
- 连续 7 天无相关内容自动降级。
- 运营可手动移出。

### 11.4 热度误判风险

风险：

- 样本不足时容易误判升温/降温。

控制：

- 所有热度结论带置信度。
- 样本不足时只输出线索，不输出确定判断。
- 输出补采建议。
- 后续逐步引入平台、行业、内容类型基线。

## 12. 第一阶段验收标准

第一阶段完成后，运营应能完成下面闭环：

1. 创建 `K12教育 + 单亲妈妈` 词包。
2. AI 生成扩展词，运营确认启用。
3. 系统按词包生成小红书每日采样任务。
4. 采集完成后自动分析文本内容。
5. 系统从内容中沉淀达人画像。
6. 达人自动进入 A/B/C 候选池。
7. A 档达人自动进入监控池。
8. 监控池按频率采集达人内容。
9. 人工添加重点友商并进入每天 3 次监控。
10. 友商采集后自动生成流量组成快照。
11. 关键词热度输出 24h/7d/30d、置信度和补采建议。
12. 运营可以在工作台查看证据、调整档位、移入/移出监控池。

## 13. 推荐实施顺序

推荐先做能打通闭环的最小链路：

1. 追踪词包结构和确认流程。
2. 词包每日采样调度。
3. 采集后自动文本分析和达人画像。
4. A/B/C 分层和 A 档自动入池。
5. 监控池分层调度。
6. 友商快照自动化。
7. 热度置信度和补采建议。

不建议先做大规模前端重构，也不建议先做复杂语义搜索。当前最重要的是让数据链路自动跑起来，让运营每天打开工作台就能看到新增内容、新增达人、自动入池、友商变化和关键词热度线索。
