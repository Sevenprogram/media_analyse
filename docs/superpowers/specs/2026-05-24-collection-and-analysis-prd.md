# 采集新数据与自动分析 PRD

## 1. 背景

当前系统已经具备以下能力：

- 页面展示数据库中的内容追踪与友商监控数据
- 后端基于数据库样本生成分析快照
- 前端读取最新分析快照与历史记录

当前缺口在于：页面只能消费`已有数据`，不能从页面侧发起`采集新数据 -> 写库 -> 分析 -> 刷新结果`的完整闭环。

本需求目标是补上这段能力。

---

## 2. 产品目标

把现有链路从：

`数据库已有数据 -> 分析 -> 页面展示`

扩展为：

`页面触发采集 -> 后端采集任务 -> 数据写库 -> 分析任务 -> 页面展示最新结果`

本期不做“页面请求时现爬现算”的同步模式，统一采用`任务化执行`。

---

## 3. 适用页面

### 3.1 内容追踪页

目标：

- 补充 tracker 对应的最新内容样本
- 基于新样本更新 tracker 分析快照
- 让用户能在页面内完成“采集并分析”

### 3.2 友商监控页

目标：

- 补充 competitor 对应账号的最新内容和互动快照
- 刷新日报、贡献榜、组成分析、异常流
- 让用户在页面内完成“同步并更新分析”

---

## 4. 核心产品原则

### 4.1 采集与分析解耦

采集任务只负责：

- 请求平台数据
- 标准化结果
- 入库

分析任务只负责：

- 基于数据库数据生成分析结果

### 4.2 页面不等待长任务完成

页面点击“采集并分析”后：

- 立即拿到 `run_id`
- 后端异步执行
- 页面轮询状态

### 4.3 允许部分成功

如果多平台采集时部分平台失败：

- 整体状态记为 `partial_failed`
- 已成功平台的内容仍然入库
- 分析任务可配置为仍然继续执行

### 4.4 风险可见

页面需要清楚提示：

- 哪个平台失败
- 失败原因
- 是否有部分结果已成功写入
- 是否已触发分析

---

## 5. 内容追踪页设计

### 5.1 新增按钮

- `采集新数据`
- `采集并分析`
- `仅重新分析`

### 5.2 行为定义

#### 采集新数据

作用：

- 根据 tracker 的平台、关键词、排除词、时间窗口补采样本
- 不强制生成新分析快照

适用场景：

- 先补数据，再人工判断是否分析

#### 采集并分析

作用：

- 先补采样本
- 采集结束后自动触发 tracker 分析
- 成功后页面刷新最新分析

适用场景：

- 想直接得到最新分析结果

#### 仅重新分析

作用：

- 不采集
- 直接基于数据库中现有样本重跑分析

适用场景：

- 调整了分析逻辑
- 样本已有，但想刷新结果

### 5.3 页面状态

#### 待执行

- 按钮可点击
- 显示上次采集/分析时间

#### 采集中

- 显示任务状态：`queued / running`
- 显示阶段：`正在采集`
- 禁止重复触发相同任务

#### 分析中

- 采集结束后切换到 `正在分析`
- 允许用户继续浏览旧分析结果

#### 成功

- 展示新增样本数
- 展示更新时间
- 刷新最新分析快照

#### 部分失败

- 展示成功平台 / 失败平台
- 展示已入库内容数
- 若分析已执行，显示分析结果已基于部分数据更新

#### 失败

- 展示失败原因
- 提供“重试”入口

### 5.4 页面输出

成功后应至少反馈：

- 本次采集新增内容数
- 更新内容数
- 影响平台数
- 是否触发分析
- 分析 run 状态

---

## 6. 友商监控页设计

### 6.1 新增按钮

- `同步最新内容`
- `同步并更新分析`

### 6.2 行为定义

#### 同步最新内容

作用：

- 采集当前 competitor 对应账号的最新内容和互动快照
- 刷新数据库
- 不强制刷新全部分析区

#### 同步并更新分析

作用：

- 先同步账号最新内容
- 再刷新 summary、ranking、composition、anomalies

### 6.3 页面状态

- 当前账号同步中
- 同步成功
- 同步部分失败
- 同步失败

要求：

- 不锁死整页
- 只局部更新当前账号数据
- 允许用户切换账号

### 6.4 页面输出

应至少展示：

- 新增内容数
- 更新快照数
- 最近同步时间
- 是否刷新了分析模块

---

## 7. 用户流程

### 7.1 内容追踪页

1. 用户进入 tracker 详情页
2. 点击 `采集并分析`
3. 前端调用后端创建采集任务
4. 页面显示运行状态
5. 采集结束后后端自动创建分析任务
6. 页面轮询任务状态
7. 成功后刷新最新分析快照

### 7.2 友商监控页

1. 用户选择一个监控账号
2. 点击 `同步并更新分析`
3. 前端调用后端创建 competitor 采集任务
4. 页面显示同步中
5. 后端完成采集并刷新相关分析数据
6. 页面刷新当前账号视图

---

## 8. 任务状态定义

统一状态：

- `queued`
- `running`
- `succeeded`
- `partial_failed`
- `failed`
- `cancelled`

推荐阶段字段：

- `collecting`
- `normalizing`
- `persisting`
- `triggering_analysis`
- `analyzing`
- `completed`

---

## 9. 失败场景与产品处理

### 9.1 平台无结果

处理：

- 标记成功但新增内容数为 0
- 不视为失败

### 9.2 平台限流 / 登录失效

处理：

- 标记平台失败
- 总体可记为 `partial_failed`

### 9.3 数据写库失败

处理：

- 标记任务失败
- 不触发分析

### 9.4 采集成功但分析失败

处理：

- 采集 run 成功
- 分析 run 失败
- 页面分别提示

---

## 10. P0 范围

本期只做：

- 内容追踪：手动 `采集新数据` / `采集并分析`
- 友商监控：手动 `同步最新内容` / `同步并更新分析`
- 任务状态轮询
- 页面状态反馈

本期不做：

- 定时任务
- 自动补采
- 自动告警
- 页面现爬现算

---

## 11. 验收标准

### 内容追踪

- 可以从页面发起 tracker 采集任务
- 采集成功后数据写入数据库
- 可以自动触发分析
- 页面能看到新的分析快照

### 友商监控

- 可以从页面发起 competitor 同步任务
- 同步成功后账号相关模块可刷新
- 失败时页面可显示明确错误

### 共性

- 页面不长时间阻塞等待
- 支持 `queued / running / succeeded / partial_failed / failed`
- 不会因为部分平台失败导致整页不可用
